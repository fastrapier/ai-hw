## Часть 1. FastAPI API для ML-модели

Для API была выбрана текстовая модель анализа тональности из `p1/text`, потому что у неё простой формат входа и выхода: на вход подаётся текст, на выходе возвращаются `label` и `score`. Это позволяет быстро и прозрачно показать идею HTTP API для ML без загрузки изображений, аудио или видеофайлов.

В `p2/app/main.py` реализованы маршруты:

- `GET /health` - проверка состояния сервиса и статуса загрузки модели.
- `POST /predict` - анализ одного текста или списка текстов.

Загрузка модели вынесена в сервис `SentimentService`, который лениво создаёт Hugging Face `pipeline("sentiment-analysis")` и затем переиспользует его между запросами.

## Часть 2. Тесты и автоматический запуск

Для API написаны тесты на `pytest` и `fastapi.testclient`.

Покрыты сценарии:

- успешный запрос к `POST /predict`;
- батчевый запрос;
- валидация пустого запроса;
- доступность маршрутов `GET /health`, `GET /llm/health`, `POST /llm/generate`, `POST /agent/answer`.

В тестах используется подмена зависимостей, поэтому CI не скачивает настоящую модель Hugging Face и не требует локального поднятия Ollama. Для автоматического прогона добавлен workflow GitHub Actions, запускающий тесты на каждом `push` и `pull_request`.

## Часть 3. API для LLM

Локальная LLM разворачивается через Ollama, как и в `p1`. Поверх нативного API Ollama в `p2` сделана собственная FastAPI-обёртка.

Реализованы маршруты:

- `GET /llm/health` - показывает доступность Ollama и список установленных моделей;
- `POST /llm/generate` - принимает `prompt`, `model`, `system`, `temperature` и возвращает единый JSON-ответ.

Такой слой удобен тем, что собственное приложение получает стабильный API-контракт и не зависит напрямую от формата запросов к Ollama во внешнем коде.

## Часть 4. Сравнение лёгких LLM

Для сравнения выбраны три модели:

- `llama3.2:3b`;
- `qwen2.5:3b`;
- `phi3:mini`.

Для них зафиксирован единый набор промптов в `p2/eval/prompts.json`. Проверяются четыре свойства:

- следование инструкции;
- базовая логика;
- устойчивость к галлюцинациям;
- полезность для агентного сценария.

Итоги сравнения сведены в `p2/eval/results.md`. После реального прогона через локальный Ollama для агентного сценария выбрана `llama3.2:3b`, так как она показала самый аккуратный ответ в тесте на галлюцинации и более надёжное поведение в режиме "отвечай только по доступному контексту".

## Часть 5. ИИ-агент на основе LLM

ТЗ агента: помочь студенту готовиться к защите проекта `p2`, отвечая на вопросы по реализованному API, выбранным моделям и требованиям задания.

Реализация:

- агент доступен через `POST /agent/answer`;
- внутри используется `StudyAgentService`;
- перед ответом агент выбирает набор простых "инструментов" - источников контекста: отчёт, README, результаты оценки моделей, текст задания;
- затем он формирует системный промпт и отправляет вопрос в выбранную модель через Ollama.

Такой подход делает поведение агента более контролируемым: он отвечает не "из головы", а с опорой на локальный контекст проекта.

## Метрики оценки моделей

Для ML API:

- корректность структуры ответа;
- средняя латентность ответа;
- стабильность повторных запросов;
- доля успешных ответов без ошибок сервера.

Для LLM и агента:

- следование инструкции;
- полезность ответа для пользователя;
- устойчивость к галлюцинациям;
- краткость и структурированность ответа;
- время генерации ответа.

## Уязвимости и ограничения

- Галлюцинации LLM: модель может придумать факт, если в контексте нет данных. Снижение риска: явная инструкция признавать отсутствие информации и ограничение контекста локальными файлами.
- Prompt injection: пользователь может попытаться заставить агента игнорировать системный промпт. Снижение риска: жёсткий системный промпт и выдача ответов только по локальному контексту.
- Недоступность Ollama: если локальный сервис не запущен, LLM-маршруты и агент не работают. Снижение риска: health-check и обработка ошибок `503`.
- Медленный CPU-инференс: лёгкие модели всё равно могут отвечать медленнее, чем облачные решения. Снижение риска: использование 3B-класса моделей и простых сценариев.
