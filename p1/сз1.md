**ПрИнж 25/26. Семестр 1. Серия заданий 1. (с1сз1)**

Цель задания: научиться создавать приложения с использованием готовых библиотек машинного обучения.

Концепт задания (это интро, а не текст самого задания):

В рамках выполнения данного задания Вам необходимо ознакомиться с возможностями ИИ/ML/DL для решения прикладных задач, определить, какие прикладные задачи можно решать с помощью библиотек и моделей машинного обучения, **выбрать для себя 4 наиболее интересные задачи + LLM** и решать их с применением существующих обученных моделей.

Необходимо

Например, Вы можете решить 4 следующие задачи:

- определение тональности текстов,
- классификация изображений;
- задача перевода аудио в текст;
- детекция объектов на видео.

Стоит определить наиболее интересные для себя варианты задач.

Обратите внимание на то, что необходимо учитывать, что для одной из четырех моделей позднее нужно будет разработать API.

Также нужно подобрать LLM, которую возможно развернуть на Вашей инфре/локально на компе и развернуть её) Вероятнее всего, это будет модель 2/3/7/8B (на созвоне поясню, если напомните).

Задание:

- Создайте репозиторий на GitHub.
- Изучите возможности готовых библиотек машинного обучения.
- Сформулируйте задачу, которую Вы хотите решить с помощью инструментов машинного обучения и кратко опишите техничское задание (ТЗ).
- Реализуйте решение выбранной Вами задачи с использованием готовой библиотеки машинного обучения.
- Загрузите реализованное решение в репозиторий на GitHub.
- Оформите документацию на Ваше решение в репозитории.
- (!) Выполнить пункты 3-6 в рамках формирования программного решения для следующих **4** задач (с разным типом контента):

1 работа с текстом (например, анализ тональности естественного языка, генерация текста, ...)

2 обработки аудио (синтез речи, speech-to-text, command detection, ...)

3 обработки изображений (классификация, обнаружение объектов на изображении, улучшение качества изображения, ...)

4 обработки видео (детекция объектов, распознавание жестов, ...)

Решения необходимо реализовывать с применением не менее чем 2ух вариантов фреймворков. (Hugging Face/PyTorch/TensorFlow)

Например:

1.обработка естественного языка (Hugging Face)

2 обработка аудио (TensorFlow)

3 обработка изображений (PyTorch)

4 обработка видео (Hugging Face)

+

1. LLM (опенсорсная, никаких особых требований нет, просто чтобы она работала локально (не на серверах сторонних провайдеров)

То есть Вы работаете только с готовыми решениями. Обучать модели/дообучать (файнтюнить) ЛЛМки не требуется - нужно только понять возможности ИИ на сегодняшний день и развернуть их у себя на компе, обертки/фронтенд также не требуется (но не возбраняется)



По результатам выполнения задания формируется Отчёт (в том числе, включающий описание

- преимуществ выбранной библиотеки,
- принципа работы Модели и причин её выбора,
- структуры датасета,
- метрик оценки качества модели (как оценить, что модель работает нормально? Можно отслеживать ML-метрики (например, Accuracy, Recall...), продуктовые метрики (Session time), инфраструктурные метрики (например, GPU)),
- особенностей реализации алгоритма, его эффективности).
