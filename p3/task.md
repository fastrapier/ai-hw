**Сз_3. Part 1. LLMOps (Observability)**

Для выполнения данного задания необходимо иметь работающего ИИ-агента (которого нужно было реализовать в рамках Cз_2)

Для того, чтобы упреждать отказы/сбои ИИ-агента, имеет смысл создавать для него соотвествующую инфру и настроить систему наблюдаемости. Как правило, базовый минимум для observability -- это логи+трейсы+метрики, но это почти для любой системы актуально, у LLM(-агентов) также есть ещё специфические «фичи», которые имеет смысл мониторить (например, версионирование промтов, расход токенов итд, и не факт, что всё это в каждом конкретном случае получится мониторить без ллм-специфичного стека для настройки наблюдаемости).

В рамках этого задания Вам предлагается настроить систему мониторинга для ИИ-агента (логи+метрики+трейсы), используя опенсорсные решения (в плане стека нет ограничений, выбирайте то, что Вам ближе), например:

\- Langtrace

\- Langfuse

\- OpenLLMetry

...

Сначала стоит сделать обзор (сравнительный анализ) минимум 3 вариантов стека для решения этой задачи (аргументируйте выбор стека).

Можно использовать как классический стек для сбора логов/метрик/трейсов/ типа Loki/Jaeger/Promrtheus ... etc, так и варианты решений, разработанных именно для ллм -- все варианты ок, выбирайте то, что наиболее релевантно для обогащения Вашего бэкграунда)
